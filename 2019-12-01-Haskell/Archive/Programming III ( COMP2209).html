You will also need to download the Haskell file

Challenges.hs

that contains the function signatures and dummy implementations as well as downloading the Haskell

Test Harness

that we will use to run our automated tests for marking and feedback. Feel free to add more of your own tests to this file. You will notice that the tests rely upon checking equality between the result of executing your solution to the challenges and some fixed expected result. Where there may be some variability in results, appropriate notions of equivalence will be defined to perform this check. These are not reflected in the Test Harness file above. You are free to vary the equality checks in your testing code as you see fit.

We have provided the file Parsing.hs that contains many of the useful functions from Chapter 13 on Monadic Parsing in Hutton's book. You will need to download this file to compile the Challenges.hs file.

At the lecture on Tuesday (5pm, 12th Nov 67/1037), Andy Gravell will answer any questions you may have about the assessment process or the functions required in the instructions themselves.

Please read the instructions before then and come along if anything is unclear.
